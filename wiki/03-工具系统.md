# 工具系统

## 概述

工具系统基于 LangChain `BaseTool`，通过 `@tool` 装饰器定义，由 `ToolRegistry` 统一管理。

## 工具注册中心

### ToolRegistry 类

```python
class ToolRegistry:
    def __init__(self):
        self._tools: dict[str, BaseTool] = {}
        self._register_default_tools()

    def register(self, tool: BaseTool) -> None:    # 注册工具
    def get(self, name: str) -> BaseTool | None:   # 获取工具
    def execute(self, name: str, args: dict) -> str # 执行工具
    def get_langchain_tools(self) -> list[BaseTool] # 获取 LangChain 格式列表
    def list_tools(self) -> list[str]               # 列出所有工具名
```

### 默认注册的工具

| 工具名 | 功能 | 文件 |
|--------|------|------|
| `read_file` | 读取文件内容 | file_tools.py |

## 工具定义示例

### 文件工具

```python
@tool
def read_file(path: str, line_numbers: bool = False) -> str:
    """读取指定路径的文件内容。

    Args:
        path: 文件路径
        line_numbers: 是否显示行号，默认 False

    Returns:
        文件内容字符串
    """
    file_path = Path(path)
    if not file_path.exists():
        raise FileNotFoundError(f"文件不存在: {path}")
    return file_path.read_text(encoding="utf-8")
```

### Shell 工具

```python
@tool
def run_command(command: str, timeout: int = 30, cwd: str | None = None) -> str:
    """执行 shell 命令并返回输出。

    Args:
        command: 要执行的命令
        timeout: 超时时间（秒），默认 30
        cwd: 工作目录，默认当前目录

    Returns:
        命令的标准输出，或错误信息
    """
    result = subprocess.run(
        command, shell=True, capture_output=True,
        text=True, timeout=timeout, cwd=cwd
    )
    return result.stdout
```

## 如何添加新工具

### 步骤 1：定义工具

在 `tools/` 目录下创建或编辑文件：

```python
# tools/my_tools.py
from langchain_core.tools import tool

@tool
def my_custom_tool(param: str) -> str:
    """工具描述（会被 LLM 看到）

    Args:
        param: 参数描述

    Returns:
        结果描述
    """
    # 实现逻辑
    return "result"
```

### 步骤 2：注册工具

在 `tools/registry.py` 的 `_register_default_tools()` 中添加：

```python
def _register_default_tools(self) -> None:
    from nano_code.tools.my_tools import my_custom_tool

    default_tools = [
        # ... 现有工具
        my_custom_tool,  # 新增
    ]
```

## 工具执行流程

```
LLM 决定调用工具
       │
       ▼
thinking_node 提取 tool_calls
       │
       ▼
execute_node 调用 registry.execute()
       │
       ▼
ToolRegistry.execute(name, args)
       │
       ▼
BaseTool.invoke(args)
       │
       ▼
返回结果字符串
```

## 最佳实践

1. **清晰的文档字符串**：LLM 通过 docstring 理解工具用途
2. **类型注解**：使用 Python 类型注解帮助 LLM 理解参数
3. **错误处理**：返回有意义的错误信息，而非抛出异常
4. **幂等性**：尽量设计可重复执行的工具
5. **安全性**：限制危险操作（如 `rm -rf`）
