# 权限系统

## 概述

权限系统为工具执行提供安全控制，包括路径隔离、命令过滤和用户确认机制。

## 架构设计

```
┌─────────────────────────────────────────────────────────────┐
│                        Agent 请求                            │
│                    "删除 src/ 目录"                          │
└─────────────────────────┬───────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                   PermissionManager                          │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │ PathGuard   │  │ CmdGuard    │  │ ConfirmGuard│         │
│  │ 路径检查    │  │ 命令过滤    │  │ 用户确认    │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
└─────────────────────────┬───────────────────────────────────┘
                          │
          ┌───────────────┼───────────────┐
          ▼               ▼               ▼
     [ALLOW]         [CONFIRM]        [DENY]
      执行            等待确认          拒绝
```

## 权限级别

| 级别 | 说明 | 行为 |
|------|------|------|
| `ALLOW` | 自动允许 | 直接执行，无需确认 |
| `CONFIRM` | 需确认 | 弹出提示，等待用户批准 |
| `DENY` | 禁止执行 | 返回错误信息 |

## 核心组件

### PermissionManager

权限管理器，协调所有守卫进行检查。

```python
from nano_code.security.manager import PermissionManager, PermissionConfig

# 创建配置
config = PermissionConfig(
    workspace_root=Path("."),
    denied_paths=[".env", "*.key"],
    denied_commands=["sudo", "rm -rf /"],
)

# 创建管理器
manager = PermissionManager(config)

# 检查权限
result = manager.check("read_file", {"path": "src/main.py"})
if result.allowed:
    # 执行工具
    pass
elif result.needs_confirm:
    # 请求用户确认
    pass
else:
    # 拒绝执行
    print(result.reason)
```

### PathGuard

路径权限守卫，控制文件系统访问。

| 功能 | 说明 |
|------|------|
| workspace 隔离 | 限制只能在指定目录内操作 |
| 白名单模式 | 只允许匹配的路径 |
| 黑名单模式 | 禁止访问敏感文件 |
| 写入确认 | 修改关键文件需确认 |

### CommandGuard

命令权限守卫，控制 Shell 命令执行。

| 功能 | 说明 |
|------|------|
| 命令白名单 | 安全命令自动执行 |
| 危险命令拦截 | `sudo`, `rm -rf /` 等 |
| 超时限制 | 防止长时间运行 |
| 网络控制 | 可禁用 `curl`, `wget` 等 |

## 配置文件

配置文件路径：`.nano-code/permissions.yaml`

### 开发模式（宽松）

```yaml
workspace:
  root: "."
  allow_outside: false

file:
  denied_paths:
    - ".env"
    - "*.pem"

shell:
  enabled: true
  default: confirm
  denied_commands:
    - "rm -rf /"
    - "rm -rf ~"
```

### 生产模式（严格）

```yaml
workspace:
  root: "/app/workspace"
  allow_outside: false

file:
  default: confirm
  allowed_paths:
    - "src/**"
    - "tests/**"
  denied_paths:
    - ".env"
    - ".git/**"
    - "secrets/**"
  confirm_on_write:
    - "**"

shell:
  enabled: true
  default: confirm
  allowed_commands:
    - "ls"
    - "cat"
    - "pytest"
  denied_commands:
    - "rm *"
    - "sudo *"

global:
  max_tool_calls: 50
  audit_log: true
```

## 使用示例

### 集成到 ToolRegistry

```python
from nano_code.security.manager import PermissionManager, PermissionConfig
from nano_code.tools.registry import ToolRegistry
from nano_code.cli.confirm import request_user_confirmation

# 初始化权限管理器
config = PermissionConfig.from_yaml(Path(".nano-code/permissions.yaml"))
permission_manager = PermissionManager(config)

# 创建工具注册表
registry = ToolRegistry(
    permission_manager=permission_manager,
    confirm_callback=request_user_confirmation,
)

# 执行工具（自动检查权限）
result = registry.execute("read_file", {"path": "src/main.py"})
```

## 审计日志

启用后，所有工具调用都会记录到 `.nano-code/audit.log`：

```json
{"timestamp": "2025-01-15T10:30:00", "tool": "read_file", "args": {"path": "src/main.py"}, "result": "allow"}
{"timestamp": "2025-01-15T10:30:05", "tool": "run_command", "args": {"command": "rm -rf /"}, "result": "deny", "reason": "命令匹配禁止模式"}
```

## 目录结构

```
src/nano_code/security/
├── __init__.py          # 模块导出
├── permission.py        # 权限级别和结果定义
├── guards.py            # 守卫基类
├── path_guard.py        # 路径守卫实现
├── command_guard.py     # 命令守卫实现
└── manager.py           # 权限管理器
```

## 安全建议

1. **生产环境**：使用严格配置，启用审计日志
2. **敏感文件**：添加到 `denied_paths` 黑名单
3. **危险命令**：扩展 `denied_commands` 列表
4. **定期审计**：检查 `audit.log` 发现异常行为
